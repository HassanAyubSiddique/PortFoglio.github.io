Diabetes

Artificial Intelligence Project

A Python solution was developed for diabetes classification using nine attributes. The project employed both a Logistic Regression model and a Random Forest model to maximise the categorisation rate. The dataset was cleaned by removing rows with missing values, ensuring the model was trained on complete data. The eight independent features (X) were used to predict the outcome variable (Y).

The Logistic Regression model utilised normalised features, transforming data to a range between -1 and 1, which enhanced model efficiency by equalising the influence of each variable. Conversely, the Random Forest model standardised features to a range between 0 and 1, balancing the weight of features with disparate scales and improving performance. The choice between normalisation and standardisation depended on the data and the model used.

The project involved selecting three key features based on their relevance scores to improve model prediction. Initially, the data was split with 70% for training and 30% for testing, with various ratios tested later for accuracy impact. Models were evaluated using metrics like accuracy, recall, F1 score, and precision. A graph showed the effectiveness of different scaling methods, revealing that Min-Max Scaling (Normalisation) yielded the best model performance, providing valuable insights into the prediction of diabetes.
